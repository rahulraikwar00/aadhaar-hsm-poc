What is Aadhaar Secure Vault?

  • Tokenization

    Sensitive Aadhaar data replaced with secure tokens (T-UUID format). Raw data never stored.

  • HSM Encryption

    Hardware Security Module integration for cryptographic operations and key management.

  • Field-level Masking

    Automatic data masking for Aadhaar numbers, emails, and phone numbers.

  • Audit Logging

    Complete audit trail of all vault operations with PostgreSQL storage.

  • Input Validation

    Aadhaar format validation, email RFC 5322 compliance, phone number checks.

  • Prometheus Metrics

    Built-in metrics for store, retrieve, delete operations with Grafana support.

Quick Start

Run the vault with Docker Compose in seconds…

Clone & Run
# Clone and start all services
git clone https://github.com/rahulraikwar00/aadhaar-hsm-poc.git
cd aadhaar-hsm-poc
docker compose up -d
Store Aadhaar Data
curl -X POST http://localhost:8000/vault/store \
  -H "Content-Type: application/json" \
  -d '{
    "aadhaar_number": "123456789012",
    "name": "John Doe"
  }'
Retrieve Data
# Get data
curl http://localhost:8000/vault/<TOKEN>

# Get masked
curl http://localhost:8000/vault/<TOKEN>/masked
Check Health
curl http://localhost:8000/health

Built for privacy first

Aadhaar Secure Vault does not store any of your sensitive data externally. All data remains on your infrastructure with tokenization and HSM-backed encryption.

FAQ

  • Is Aadhaar Secure Vault production ready? +

    This is an open-source proof-of-concept. It is not intended for production use without further security reviews and hardening.

  • Does it require HSM hardware? +

    The POC supports both SoftHSM2 (for testing) and hardware HSMs (for production). You can start with SoftHSM and migrate later.

  • Is my Aadhaar data stored anywhere? +

    No. Raw Aadhaar data is tokenized immediately and only tokens are stored. The original data cannot be recovered from storage.

  • What about data privacy compliance? +

    The vault is designed to help with Aadhaar data privacy compliance by never storing raw data. However, you should consult with legal experts for specific compliance requirements.
